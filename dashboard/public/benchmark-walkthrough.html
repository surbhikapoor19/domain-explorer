Reading the Benchmarks page

This page is not a leaderboard. It answers one question well — “do independent papers agree when measured under the same conditions, and can I verify every number myself?” Work through the seven steps below; each has an annotated mockup of the real interface.

1 What this page is for

In robotics, a raw success-rate number means almost nothing on its own. The same method scores differently depending on a dozen things the paper chose. So the page refuses to rank globally and instead surfaces agreement under matched conditions and where the field hasn't measured yet.

Why “who's #1?” is the wrong question

Two papers reporting “92% success” and “61% success” are usually not comparable — they almost never ran the same setup. Every number is confoundedConfound: a hidden variable that changes the result independently of the method's quality — so two numbers can't be compared unless it's held constant. by all of these:

The objects

Which object set (YCB? a custom pile?), how many, how cluttered the scene.

The hardware

Gripper (two-finger vs multi-finger), arm, and the depth/RGB sensor used.

The protocol

Sim vs real, what counts as “success,” and how many trials were run.

The reframe Because of those confounds, the page never computes a global rank or a cross-condition average. It groups numbers into cells — one per (metric × condition) — and only ever compares within a cell, where the setup is matched. The whole UI is built to keep that promise.

The three things the page gives you

① Agreement

Per cell: do the papers agree, disagree, or is there only one source? With a plain-language spread.

② Coverage

A condition×metric grid where the empty cells show you exactly what nobody has tested yet.

③ Provenance

Every number deep-links to a crop of the exact source table, plus a “why they differ” explanation.

Read it like this A ✓ agree with a tight spread is a number you can lean on. A ⚠ contested row is a flag to go read why (step 6), not a tie-breaker to pick a winner. An ○ single-source row isn't “unverified bad” — it just means nobody has independently re-measured it yet (a coverage opportunity).

Why this matters Filtering isn't cosmetic here — it's how you earn a fair comparison. Two numbers are only comparable once they share a cell, and the spine/composer is the tool that puts them there.

Read down, not across
You only ever read down one matched column. Cross-cell comparison is structurally impossible here by design — that's the whole point.
Reporting gaps too Below the grid sits a traffic-light companion (rows = papers × columns = confound axes). It marks which confounds each paper actually disclosed — e.g. “is the gripper even reported?” A missing tile is a reporting gap, distinct from an untested gap above.

Reproducibility card
One per method — the IEEE/GRASPA/SceneReplica record schema: object set, gripper, arm, sensor, scene, success criterion, trials, protocol. Fields nobody extracted read not reported (never invented). It also lists explicit do-not-compare-against setups.
Source tags

Replication tier
A label like reproduced ≥2 papers / single-paper full-protocol. It is a replication signal, not a quality rank.
Provenance crop
The actual image of the source table with the cell highlighted, the caption, the paper, and a page deep-link. This is where “trust the number” becomes “see the number.”

6 Why two papers differ — the KG panel

A contested cell raises the question “why?” The drawer's “Why they differ” section answers it from the Knowledge Graph — joining each cell to its methods' attributes and its papers' relations. Every fact is source-tagged; nothing is invented.

One row per differing axis — gripper, sensor, backbone — each value shown for both methods. If the sensors differ, that is a reason two numbers diverge.
Technique lineage
Derived from the graph: shared backbones and builds_on citations (“B builds-on A”). Two methods sharing a backbone vs. using different ones is a real explanatory signal.
Citation stance
From the cites.sentiment field — e.g. one paper differs-from or builds-on another. It's a relation/claim, labeled as such, not measured truth.
Outperforms (stated only)
Only claims the paper actually stated, shown with its own evidence text — no fabricated strength number. Predicted (HGT) links are gated off until those metrics are trusted, so this panel never shows a guessed edge.
The honest line “Why they differ” explains a disagreement with grounded, sourced context — it does not pick a winner. Method-typical attributes (the method's usual rig) are tagged differently from as-run facts (what the paper actually reported), so you never mistake one for the other.

7 Asking the copilot

The same copilot you know from Graph Reasoning works here too — but on this page it is grounded only in the page's own data (the cells + their crops). Ask in natural language; it reshapes the page rather than free-typing an answer.

You ask“Which method wins for grasp success on piled scenes?”
Copilot applied: grasp-successscene · pile
On pile scenes the two reporting papers disagree (78 vs 40, differ 38 pts) — there is no clean winner, and the cell is grade C. The values diverge on sensor (TSDF vs point-cloud). cell: grasp-success·pile ↗ crop p.7 ↗
Draft before apply
A NL query becomes editable condition chips in a “Copilot applied: …” banner. The view never moves silently — you click Apply to scope the table, or Dismiss.
Cites only retrieved cells
Every claim carries a cell + crop citation. It computes out of the same shared cell data the page uses — never a parallel computation, never a web fact.
“No matched comparison”
If nothing in the corpus matches your conditions, the answer is literally “no matched comparison available.” The gap is the honest answer — it won't fabricate one.
Deep-links back
Answers emit a “View on Benchmarks →” permalink to the matching cell, and you can select a cell/column and “ask about selection.”
What it will not do It won't crown a global #1, won't average across conditions, and won't use the (currently distrusted) HGT-predicted links. If you ask “who's best overall?”, the honest answer is that the question doesn't have one — only per-cell comparisons do.

★ Honesty principles

Everything above serves these invariants. If you remember nothing else, remember these:

The page's promises

  • Every number is traceable — it deep-links to a crop of its exact source table and page.
  • Agreement is not a win. ✓ means papers concur under matched conditions; it doesn't crown a method.
  • Comparisons happen only within a cell, where the conditions are matched.
  • An empty cell is a gap (opportunity), never a zero.
  • Low-quality rows are flagged, not deleted — quarantined records are counted and revealable.
  • No global rank. No cross-condition average. No pooled mean diamond.
  • No invented facts — unextracted fields read “not reported”; distrusted predicted links stay gated off.
In one sentence The Benchmarks page trades the false comfort of a leaderboard for something a researcher can actually use: matched, verifiable, gap-aware evidence.
